UUID GENERATION METHODS
================================================================================

UUID v1 (Time-based)
- Uses: Current timestamp + random bytes
- Format: 8-4-4-4-12 hexadecimal
- Deterministic: No (includes random component)
- Use case: Unique identifiers when time-ordering is needed

UUID v4 (Random)
- Uses: crypto.getRandomValues() API
- Format: 8-4-4-4-12 hexadecimal with version/variant bits
- Deterministic: No (purely random)
- Use case: Most common; maximum randomness

UUID v5 (Name-based SHA-1)
- Uses: SHA-1 hash of namespace + name
- Format: 8-4-4-4-12 hexadecimal
- Deterministic: Yes (same input = same UUID)
- Namespaces: DNS, URL, OID, X.500 DN
- Use case: Reproducible IDs from input values

UUID v7 (Time-based Sortable)
- Uses: Unix epoch timestamp (48 bits) + random bits
- Format: 8-4-4-4-12 hexadecimal
- Deterministic: No (includes random)
- Sortable: Yes (chronological ordering)
- Use case: Modern, database-friendly UUIDs
